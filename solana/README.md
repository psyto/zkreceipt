# zkreceipt-verifier (Solana Anchor program)

Verifies Groth16 proofs of source-chain finality (generated by
[`zkreceipt-prover`](../prover/)) and persists the resulting
`(slot, state_root)` to a PDA. Downstream programs — e.g.
[`mppsol_cpi`](https://github.com/mppsol/cpi) — read the PDA + verify
Merkle Patricia inclusion proofs to authorize Solana-side settlement of the
source-chain payment. Finality is proven per source chain (Tempo/Simplex
implemented first, Ethereum sync-committee next); this verifier is
source-independent — it checks whatever Groth16 proof the prover commits.

## Status

**Partial.** Anchor program structure, account layouts, and the `bootstrap` +
`update_light_client` instruction bodies (state-binding, slot + validator-set
checks) are live. **Groth16 proof verification is real** — wired to
[`sp1-solana`](https://github.com/succinctlabs/sp1-solana)'s `verify_proof`
over the committed public output via `alt_bn128` syscalls (~200K CU), before
any state mutation. Host tests confirm it runs natively and fail-closes.

The embedded `ZKRECEIPT_VKEY_HASH` is a **placeholder** until the guest's real
verification key is generated (its `verify_update` leaf is incomplete), so the
deployed default rejects every proof — fail-closed. Remaining (Stage 2):

1. Build the guest → extract the real vkey → replace the placeholder.
2. SP1 version: `sp1-solana` embeds the v5.0.0 vk, so the guest must be built
   with SP1 v5.0.0.
3. Positive-path E2E test with a real guest proof + devnet deploy.

## Important: nested workspace

This directory is a **nested Anchor workspace** — it is *not* a member of
the outer zkReceipt Cargo workspace (the root `Cargo.toml` excludes it).
Build via Anchor's tooling, not the outer `cargo build`:

```bash
cd solana/
anchor build
```

**Toolchain requirements:**

- Anchor 0.32.1
- Solana CLI 3.1.14+ (ships platform-tools v1.52 with rustc 1.89 — supports
  `edition2024`, which recent transitive deps like `indexmap >= 2.14` require).
  Install via:
  ```bash
  agave-install init 3.1.14
  ```
  Older Solana CLI versions (2.2.x) bundle Cargo 1.79 and will fail to build
  with errors like `feature 'edition2024' is required`.
